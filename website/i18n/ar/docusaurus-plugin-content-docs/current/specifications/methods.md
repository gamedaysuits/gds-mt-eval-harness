# الواجهة المشتركة للطرائق

> **ملخص تنفيذي.** تحدد هذه الصفحة بروتوكول `TranslationMethod` الذي يجب على جميع طرائق Arena تنفيذه، وفئات الطرائق الست (`raw-llm`، `coached-llm`، `pipeline`، `custom-plugin`، `api`، `human`)، وصيغة الإضافة (plugin) الخاصة بالطريقة، و**فئات التبعيات** (S/O/A1/A2/X) التي تحدد ما إذا كان يمكن تشغيل الطريقة في بيئة التقييم المعزولة (sandbox) وتأهلها للجوائز. أي نهج ينفذ هذا البروتوكول يمكن قياس أدائه؛ وما تعتمد عليه الطريقة يحدد أين يمكنها المنافسة.

يتشارك نظام التقييم (eval harness) و champollion مفهومًا مشتركًا هو **طريقة الترجمة**. الطريقة هي أي إجراء يأخذ نصًا مصدريًا وينتج نصًا مترجمًا — سواء كان استدعاءً مباشرًا لنموذج لغوي كبير (LLM)، أو خط معالجة متعدد المراحل، أو واجهة برمجة تطبيقات من طرف ثالث، أو مترجمًا بشريًا.

## البنية المعمارية

```
Method Plugin (v2 Spec)
├── method.json           ← Manifest (name, class, entry_point, dependencies, metadata)
├── method_card.json      ← Leaderboard description (what, not how)
├── pipeline.py           ← Python module implementing TranslationMethod
└── (optional helpers)    ← Additional Python modules
```

يتم التحميل عبر `--method path/to/dir`. لا يكتشف النظام أي شيء تلقائيًا.

## نظامان، واجهة واحدة

| | Eval Harness | champollion |
|---|---|---|
| **اللغة** | Python | Node.js |
| **نقطة الدخول** | `translate.py` | `translate.js` |
| **الواجهة** | بروتوكول `TranslationMethod` | إعدادات `methodPlugin` |
| **الغرض** | تقييم دفعي مع تسجيل النقاط | توطين مباشر في بيئات التطوير/CI |
| **المخرجات** | بطاقة تشغيل مع المقاييس | ملفات لغوية مترجمة |

الطريقة التي تدعم النظامين معًا توفر نقطتي دخول — واحدة لكل بيئة تشغيل لغوية. أما **بطاقة الطريقة** فهي حلقة الوصل: إذ تصف الطريقة بصيغة يفهمها النظامان.

## بطاقة الطريقة

تصف بطاقة الطريقة *ماهية* طريقة الترجمة دون الكشف عن تفاصيل احتكارية مثل موجه النظام (system prompt) الكامل. وهي تجيب عن الأسئلة التالية:

- ما فئة هذه الطريقة؟ (نموذج لغوي خام، نموذج لغوي موجَّه، خط معالجة، واجهة برمجة تطبيقات، إلخ.)
- ما الأدوات التي تستخدمها؟ (محلل FST، قاموس، إلخ.)
- هل التنفيذ مفتوح المصدر؟
- ما الأزواج اللغوية التي تدعمها؟

راجع [مواصفات بطاقة الطريقة](/docs/specifications/methods#method-card) للاطلاع على مخطط JSON الكامل.

### مثال

```json
{
  "method_id": "fst-gated-v8",
  "name": "FST-Gated Coached Translation v8",
  "class": "pipeline",
  "description": "LLM translation with morphological validation. Failed words are retried with FST feedback.",
  "author": "Curtis Forbes",
  "tools_used": ["HFST morphological analyzer", "Wolvengrey dictionary"],
  "open_source": false,
  "dependency_class": "A2",
  "supported_pairs": ["eng>crk"]
}
```

يلخص الحقل `dependency_class` ما تحتاجه الطريقة للتشغيل والنقل — راجع [صلاحية الطريقة وفئات التبعيات](#method-validity-and-dependency-classes) أدناه.

### فئات الطرائق

| الفئة | الوصف |
|-------|-------------|
| `raw-llm` | استدعاء مباشر لنموذج لغوي كبير بحد أدنى من التعليمات |
| `coached-llm` | نموذج لغوي كبير مع موجه منظم وأمثلة وقيود |
| `pipeline` | خط معالجة متعدد المراحل بمكونات حتمية |
| `custom-plugin` | عملية خارجية تنفذ بروتوكول `TranslationMethod` |
| `api` | واجهة برمجة تطبيقات للترجمة من طرف ثالث (Google Translate، DeepL، إلخ.) |
| `human` | ترجمة بشرية (لإنشاء خطوط الأساس) |

## صلاحية الطريقة وفئات التبعيات

لا تكون الطريقة قابلة للتشغيل أو النقل إلا بقدر توافر أقل تبعياتها توافرًا. تعتمد آليتان من آليات Arena على معرفة ما تحتاجه الطريقة بدقة:

1. **التقييم في بيئة معزولة** ([مواصفات Benchmark §8.2](/docs/specifications/benchmark)) — تأتي النتائج الرسمية المعتمدة (gold-standard) من بيئة معزولة سياسة الشبكة فيها هي **الرفض الافتراضي (default-deny)**. الطريقة التي تتطلب خدمة خارجية بشكل خفي لا يمكنها إنتاج نتيجة رسمية.
2. **نقل الجائزة** ([مواصفات الجوائز](/docs/specifications/prizes)) — تُنقل الطرائق الفائزة بالجوائز إلى منظمة الحوكمة التابعة لمجتمع اللغة. الطريقة التي تتضمن محتوى لم يكن للمُقدِّم الحق في تضمينه لا يمكن نقلها بشكل قانوني. يجب أن يمتلك المُقدِّم (أو يُمنح) حقوق كل ما هو موجود داخل الحزمة.

ولجعل كلا الفحصين آليين بدلًا من اجتهادات حالة بحالة، تعلن كل طريقة عن **فئة تبعيات**، مشتقة من **بيان تبعيات** في `method.json`.

> **ملاحظة حول التسمية.** *فئة الطريقة* (في القسم أعلاه: `raw-llm`، `pipeline`، …) تصف *كيف تترجم الطريقة*. أما *فئة التبعيات* (هذا القسم) فتصف *ما تحتاجه الطريقة للتشغيل والنقل*. وهما محوران مستقلان: يمكن لطريقة من نوع `pipeline` أن تنتمي إلى أي فئة تبعيات.

### فئات التبعيات الخمس

| الفئة | الاسم | التعريف | قابلة للتشغيل في البيئة المعزولة؟ | مؤهلة للجائزة؟ |
|-------|------|-----------|-------------------|-----------------|
| **S** | مكتفية ذاتيًا | جميع الأكواد والبيانات والنماذج والأوزان تُشحن داخل دليل الطريقة، بموجب تراخيص تسمح بإعادة التوزيع والنقل المجتمعي. | ✅ نعم، كما هي | ✅ نعم |
| **O** | خارجية مفتوحة | تعتمد على مكونات مستضافة خارجيًا بموجب تراخيص مفتوحة تسمح بإعادة التوزيع (بما في ذلك تراخيص الحقوق المتروكة copyleft مثل AGPL) — مثل FST يتم تنزيله عند التثبيت. | ✅ نعم — المكونات تُثبَّت على نسخة محددة **وتُنسخ داخل التقديم** | ✅ نعم، بشروط توافق الترخيص: تُحفظ شروط الحقوق المتروكة عبر النقل، ويحصل المجتمع على الحقوق نفسها التي يمنحها الترخيص للجميع |
| **A1** | تعتمد على API، قابلة للاستبدال | تتطلب استدلال LLM في وقت التشغيل، حيث يكون النموذج **إعدادًا قابلًا للاستبدال** — يمكن إدخال أي نموذج بقدرة كافية. تكمن قيمة الطريقة في موجهاتها وبيانات التوجيه والأكواد الخاصة بها، لا في نموذج مزود بعينه. | ⚠️ فقط عبر **بوابة LLM** التي تحددها مواصفات البيئة المعزولة (🔲 مخطط لها — انظر أدناه) | ⚠️ مشروط — انظر أدناه |
| **A2** | تعتمد على API، غير قابلة للاستبدال | تتطلب استدعاءات في وقت التشغيل إلى واجهة برمجة تطبيقات خارجية لبيانات أو خدمة لا يمكن نسخها أو استبدالها — عادةً لأن المحتوى المقدم احتكاري أو غير مرخَّص (مثل واجهة برمجة تطبيقات لقاموس لا يملك القاموس الأساسي فيها ترخيصًا عامًا). | ❌ لا — لا يمكن للتبعية أن توجد في البيئة المعزولة دون إذن صاحب الحقوق | ❌ ليس قبل أن يمنح صاحب الحقوق أذونات التضمين في البيئة المعزولة **و** النقل معًا. مسموح بها في لوحة المتصدرين المفتوحة (شريحة التطوير) مع علامة **"تبعية خارجية"** ظاهرة |
| **X** | مغلقة | تتضمن محتوى لا يملك المُقدِّم حق إعادة توزيعه — مجموعات بيانات غير مرخصة، أو محتوى احتكاري مستخرج بالكشط، أو مكونات غير متوافقة الترخيص. | ❌ | ❌ غير مقبولة في أي مسار. تضمين محتوى دون امتلاك حقوقه يُعد انتهاكًا للترخيص بغض النظر عن مكان تشغيل الطريقة |

**الفئة الفعلية.** فئة تبعيات الطريقة هي الفئة *الأكثر تقييدًا* بين جميع تبعياتها المعلنة، بالترتيب S < O < A1 < A2 < X. قاموس واحد غير مرخص يجعل خط معالجة مكتفيًا ذاتيًا بخلاف ذلك من الفئة A2 (إذا تم الوصول إليه في وقت التشغيل) أو الفئة X (إذا ضُمِّن دون حقوق).

### التمييز بين A1 و A2: القابلية للاستبدال

معظم الطرائق تستدعي نماذج لغوية كبيرة. ولا تتظاهر Arena بخلاف ذلك — لكنها تميّز بين نوعين مختلفين تمامًا من الاعتماد على واجهات برمجة التطبيقات:

- **A1 (قابلة للاستبدال):** توفر واجهة برمجة التطبيقات استدلال LLM كسلعة عامة. معرّف النموذج هو مجرد إعداد: يجب أن تعمل الطريقة من البداية إلى النهاية مع أي نقطة استدلال متوافقة، بما في ذلك نموذج مفتوح الأوزان مستضاف مجتمعيًا. قد تختلف جودة المخرجات بين النماذج — وذلك على مسؤولية المطوّر، وترتبط النتائج الرسمية بالنموذج المثبَّت المستخدم في التقييم. الطريقة التي تعتمد على **حالة لدى المزود** (نموذج مضبوط دقيقًا مستضاف لدى المزود فقط، أو مخازن ملفات لدى المزود، أو مساعدين خاصين بالمزود) *ليست* قابلة للاستبدال: لا يمكن استبدال تلك الحالة، فتكون التبعية A2 ما لم تُضمَّن الأوزان أو البيانات الأساسية في التقديم.
- **A2 (غير قابلة للاستبدال):** تقدم واجهة برمجة التطبيقات شيئًا فريدًا — عادةً بيانات احتكارية أو غير مرخصة. لا يمكن لأي نقطة بديلة توفيره، ولا يمكن نسخ المحتوى إلى البيئة المعزولة دون إذن صاحب الحقوق. تعمل الطريقة في لوحة المتصدرين المفتوحة (مع علامة)، لكنها لا تستطيع إنتاج نتائج رسمية من البيئة المعزولة ولا التأهل للجوائز إلى أن توجد الأذونات.

**ما الذي ينقله فوز طريقة A1 بالجائزة فعليًا.** لا يحصل المجتمع على النموذج — فلا أحد يستطيع نقل أوزان Anthropic أو Google أو OpenAI. يغطي النقل الوصفة الكاملة *المحيطة* بالنموذج: جميع الموجهات وبيانات التوجيه وأكواد خط المعالجة ومنطق إعادة المحاولة والإعدادات ومتطلبات النموذج الموثقة. ولأن النموذج قابل للاستبدال بحكم التصميم، يمكن للمجتمع توجيه الطريقة المنقولة إلى أي مزود يختاره — أو إلى نموذج مفتوح الأوزان على أجهزته الخاصة — دون تدخل المطوّر. الوصفة مملوكة؛ أما المحرك فمستأجر وقابل للاستبدال.

### بيان التبعيات (`method.json`)

تعلن كل طريقة عن تبعياتها في بيان `method.json`. يسجل كل مُدخل ماهية المكوّن، ومصدره، والترخيص الذي يغطيه، وكيفية وصول الطريقة إليه:

```json
{
  "name": "FST-Gated Coached Translation v8",
  "method_id": "fst-gated-v8",
  "class": "pipeline",
  "entry_point": "pipeline:PipelineMethod",
  "supported_pairs": ["eng>crk"],
  "dependency_class": "A2",
  "dependencies": [
    {
      "id": "giellalt-lang-crk-fst",
      "kind": "software",
      "license": "AGPL-3.0-or-later",
      "access": "mirrored",
      "source": "https://github.com/giellalt/lang-crk",
      "pin": "sha256:3f1a…",
      "redistributable": true,
      "transferable": true
    },
    {
      "id": "llm-inference",
      "kind": "model",
      "license": "proprietary",
      "access": "gateway",
      "source": "openrouter:google/gemini-2.5-flash",
      "substitutable": true,
      "redistributable": false,
      "transferable": false,
      "notes": "Any compatible chat-completions endpoint works; the model slug is configuration."
    },
    {
      "id": "crk-dictionary-api",
      "kind": "service",
      "license": "none",
      "access": "external-api",
      "source": "https://itwewina.altlab.app/",
      "redistributable": false,
      "transferable": false,
      "notes": "Dictionary content has no public license; runtime lookups only. Class A2 until the rights holders grant permission."
    }
  ]
}
```

| الحقل | مطلوب | الوصف |
|-------|----------|-------------|
| `id` | ✅ | معرّف ثابت للتبعية |
| `kind` | ✅ | `data` أو `model` أو `software` أو `service` |
| `license` | ✅ | معرّف SPDX، أو `proprietary`، أو `none`. تعني `none` عدم وجود ترخيص عام — وتُعامل على أنها جميع الحقوق محفوظة |
| `access` | ✅ | `bundled` (يُشحن في دليل الطريقة)، `mirrored` (يُجلب عند التثبيت، مثبَّت على نسخة محددة، ومضمَّن في التقديم)، `gateway` (استدلال LLM في وقت التشغيل عبر بوابة التقييم)، `external-api` (أي استدعاء شبكي آخر في وقت التشغيل) |
| `source` | ✅ | عنوان URL قياسي أو معرّف `provider:slug` |
| `pin` | لـ `mirrored` | إصدار أو commit أو تجزئة محتوى تثبّت المكوّن بدقة |
| `substitutable` | لـ `gateway`/`external-api` | ما إذا كان يمكن لأي نقطة نهاية متوافقة تقديم هذه التبعية |
| `redistributable` | ✅ | ما إذا كان الترخيص يسمح بإعادة توزيع المكوّن |
| `transferable` | ✅ | ما إذا كان يمكن نقل المكوّن (أو حقوقه) إلى مجتمع بموجب شروط نقل الجائزة |
| `notes` | ❌ | سياق حر الصياغة |

**اشتقاق الفئة.** تسهم كل تبعية بفئة؛ وتكون `dependency_class` الخاصة بالطريقة هي الأكثر تقييدًا:

| ملف التبعية | يسهم بـ |
|--------------------|-------------|
| `bundled` + ترخيص يسمح بإعادة التوزيع والنقل | S |
| `mirrored` + ترخيص مفتوح يسمح بإعادة التوزيع (بما في ذلك الحقوق المتروكة) | O |
| `gateway` + `substitutable: true` (استدلال LLM) | A1 |
| `external-api`، أو `gateway` مع `substitutable: false` | A2 |
| `bundled` + `license: none` أو ترخيص غير متوافق مع إعادة التوزيع | X |

يجب أن تتطابق `dependency_class` المعلنة مع الفئة التي يشتقها النظام من البيان. وأي عدم تطابق يُعد خطأ في التحقق.

الطريقة التي **لا** تملك أي تبعيات خارجية تعلن `"dependency_class": "S"` و `"dependencies": []`. والمصفوفة الفارغة بيان إيجابي مؤكد، يُدقَّق مثل أي بيان آخر.

### كيفية التحقق من الصلاحية

ثلاث طبقات، من الأقل تكلفة إلى الأكثر حجية:

1. **تدقيق البيان.** يشتق النظام الفئة الفعلية من البيان ويرفض حالات عدم التطابق. يتحقق المراجعون من كل تبعية معلنة مقابل ترخيصها ومصدرها المعلنين — فالتبعية المعلنة `redistributable: true` التي يقول ترخيصها الأصلي خلاف ذلك تفشل في المراجعة.
2. **التحليل الساكن.** يُفحص الكود المقدم بحثًا عن استدعاءات شبكية وتنزيلات ديناميكية ووصول إلى نظام الملفات لا يفسرها البيان. والعثور على تبعية *غير معلنة* أثناء المراجعة سبب للرفض بغض النظر عن الفئة التي كانت ستنتمي إليها — يجب أن يكون البيان كاملًا، لا دقيقًا فحسب.
3. **سياسة الشبكة للبيئة المعزولة.** تتطلب مواصفات البيئة المعزولة **رفضًا افتراضيًا للاتصالات الصادرة (default-deny egress)**: لا تحصل حاويات الطرائق على أي وصول شبكي ما لم يُدرَج مسار صراحةً في قائمة السماح. المسار الصادر الوحيد الذي تحدده المواصفات هو **بوابة LLM** — وكيل استدلال تديره بنية التقييم التحتية، مقيد بقائمة سماح صريحة من النماذج المثبَّتة، مع تسجيل كل طلب واستجابة للتدقيق بعد التشغيل. أي شيء غير مدرج في قائمة السماح يفشل على مستوى الشبكة، لا على مستوى السياسة. راجع [مواصفات Benchmark §8.6](/docs/specifications/benchmark) للاطلاع على سياسة الشبكة وتصميم البوابة.

> 🔲 **مخطط لها.** البيئة المعزولة وبوابة LLM الخاصة بها محددتان بالمواصفات لكنهما لم تُبنيا بعد. وإلى أن تصبح البوابة عاملة، لا يمكن تقييم سوى طرائق الفئتين S و O في البيئة المعزولة؛ أما طرائق الفئة A1 فهي مؤهلة للجوائز *من حيث المبدأ* لكنها لا تستطيع بعد إنتاج نتائج رسمية معتمدة. تصف هذه الصفحة ما تتطلبه المواصفات، لا ما يعمل حاليًا.

### العرض في لوحة المتصدرين

- تعرض لوحة المتصدرين فئة التبعيات لكل طريقة إلى جانب شارة فئة الطريقة.
- تحمل طرائق الفئة A2 في لوحة المتصدرين المفتوحة علامة **"تبعية خارجية"** ظاهرة: إذ تعتمد نتائجها على خدمة طرف ثالث قد تتغير أو تختفي، وهي غير مؤهلة للجوائز حاليًا.
- لا تُدرج طرائق الفئة X.

## Eval Harness: بروتوكول TranslationMethod

يستخدم نظام التقييم الكتابة البنيوية في Python (`Protocol`) للإضافات. أي فئة (class) بتوقيع الدالة الصحيح تعمل — دون الحاجة إلى وراثة:

```python
class MyMethod:
    async def translate(self, entries: list[dict], config: RunConfig) -> list[dict]:
        results = []
        for entry in entries:
            translation = await self.do_translation(entry["source"])
            results.append({
                "id": entry["id"],
                "predicted": translation,
                "latency_s": 0.5,
                "usage": {"prompt_tokens": 0, "completion_tokens": 0},
                "error": None,
                "tool_calls": [],
                "tool_call_count": 0,
                "metadata": {},
            })
        return results
```

راجع [بروتوكول الإضافات](/docs/specifications/methods#eval-harness-translationmethod-protocol) للتوثيق الكامل بما في ذلك أمثلة الأغلفة (wrappers) للطرائق غير المكتوبة بلغة Python.

## champollion: إعدادات methodPlugin

في champollion، تُسجَّل الطرائق لكل زوج لغوي في `champollion.config.json`:

```json
{
  "version": 3,
  "pairs": {
    "en:crk": {
      "methodPlugin": "crk-coached-v1"
    }
  }
}
```

راجع [مواصفات الإضافات](https://champollion.dev/docs/reference/plugin-spec) لواجهة الإضافات من جانب champollion.

## التكامل مع لوحة المتصدرين

عند إرفاق بطاقة طريقة بتشغيل (عبر `--method-card`)، تُضمَّن في بطاقة التشغيل وتُعرض في لوحة المتصدرين:

```bash
# Run with method card attached
mt-eval run \
  --method path/to/my-method \
  --corpus data/edtekla-dev-v1.json \
  --method-card method_card.json

# Publish to the leaderboard
mt-eval publish eval/logs/harness/your-run-card.json
```

إذا لم يتم توفير `--method-card`، يطلق `mt-eval publish` معالجًا تفاعليًا يرشدك خلال وصف طريقتك.

تعرض لوحة المتصدرين:
- **شارة الفئة** — مؤشر بصري (مثل "pipeline"، "coached-llm")
- **فئة التبعيات** — S/O/A1/A2 (راجع [صلاحية الطريقة وفئات التبعيات](#method-validity-and-dependency-classes))؛ تحمل طرائق A2 علامة "تبعية خارجية"
- **اسم الطريقة** — من بطاقة الطريقة
- **الأدوات المستخدمة** — مدرجة من بطاقة الطريقة
- **مؤشر المصدر المفتوح**

عند عدم إرفاق بطاقة طريقة، تعرض لوحة المتصدرين الإعدادات الأصلية للنظام (النموذج، إصدار الموجه، درجة الحرارة، الأدوات المفعّلة).

:::danger ممنوع التدريب على بيانات التقييم
الطرائق التي تضمنت عملية تطويرها التعرض لمجموعة بيانات التقييم — سواء كبيانات تدريب، أو أمثلة few-shot، أو مدخلات قاموسية، أو مواد لضبط الموجهات — سيتم **استبعادها** من لوحة المتصدرين. راجع [تقييم الترجمة الآلية](/docs/leaderboard/rules) لمعرفة ما يميز الطريقة الجيدة عن السيئة.
:::
