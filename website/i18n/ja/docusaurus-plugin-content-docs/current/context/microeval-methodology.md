# Microeval：機械翻訳のための言語固有評価指標

***FST、辞書、言語学者が精選した等価規則を用いて個別言語に特化した評価指標を構築するための方法論 — そしてこの分野がそれを必要とする理由***

---

> *「私の言語の限界は、私の世界の限界を意味する。」*
> — ルートヴィヒ・ウィトゲンシュタイン、『論理哲学論考』（1921年）

---

## はじめに

機械翻訳評価のコミュニティは、20年にわたって普遍的な指標の探求を続けてきました。すなわち、あらゆる言語、あらゆるドメイン、あらゆる類型論に対して機能する翻訳品質の尺度です。この探求は、BLEU（Papineni et al., 2002）、chrF++（Popović, 2017）、COMET（Rei et al., 2020）、MetricX（Juraska et al., 2023）といった優れたツールを生み出しました。WMT共有タスクで主流となっている約20言語については、これらのツールは機能します。

残りの約7,000言語については、機能しません。

本稿は、**低リソース言語や形態論的に複雑な言語に普遍的指標を適用しようとする探求は、単に不完全なだけでなく、そもそもパラダイムとして誤っている**と主張します。私たちは **microeval** を提案します。これは、有限状態トランスデューサ（FST）、対訳辞書、形態素解析器、言語学者が精選した等価規則といった、利用可能な最良の言語ツールを用いて、個別言語に特化した評価指標を構築するための方法論です。

Microevalは指標ではありません。それは*実践*です。言語固有の知識を符号化した評価インフラを構築するための体系的なプロセスです。この実践は指標を生み出し、私たちはそれを **LYSS**（Linguistically-informed Yield & Structural Scoring）というフレームワーク名のもとに収集しています。しかし、貢献するのは方法論であり、それが生み出す特定の指標ではありません。

本文書は以下のドキュメントの補足資料です：
- [測定不可能なものを測る](/docs/context/mt-evaluation-landscape) — 既存指標の中でLYSSを位置づける評価状況の調査
- [スコアリング仕様](/docs/specifications/scoring) — 指標定義、重み付け、複合スコアリングの技術仕様
- [コーパスパートナーシップ戦略](/docs/specifications/corpus-partnership) — 評価コーパスを確立するための実践的なワークフロー

これらのドキュメントはLYSSが*何であるか*、そして*どこに位置づけられるか*を説明しています。本稿はより根本的な問いに答えます。言語固有の評価が*なぜ*必要なのか？新しい言語に対してそれを*どのように*構築するのか？そして「正しい」とは*誰が*決めるのか？

## 第1部：普遍的指標が低リソース言語で失敗する理由

### 1.1 普遍性の仮定

BLEUに始まる主要なMT評価指標はすべて、暗黙の仮定に基づいています。翻訳品質の*メカニズム*は言語に依存しないという仮定です。たとえ*パラメータ*が異なっていても。BLEUはn-gramの重複を数えます。chrF++は文字n-gramの重複を数えます。COMETは人間の判断に基づいて回帰モデルを訓練します。いずれも、翻訳が「良い」とはどういうことかを示すシグナルの構造が、言語に依存しないアルゴリズムで捉えられると仮定しています。場合によっては言語固有のデータでファインチューニングされることもあります。

高リソースのヨーロッパ言語ペアについては、この仮定は十分に成立します。WMTの指標共有タスクは、英語↔ドイツ語、英語↔チェコ語、英語↔中国語において人間との高い相関を示しています。指標はエッジケースでは意見が分かれますが、品質の分布については一致しています。

Plains Cree（nêhiyawêwin）のような多合成語言語では、この仮定は複数のレベルで崩壊します：

**形態論的不透明性。** Creeの動詞一語が、英語の節全体と同等の情報を含むことがあります。*nikî-wîcihâw*（「私は彼/彼女を助けた」）という語は、人称、数、有生性、方向、時制を一つの活用形に符号化しています。n-gram指標は一つのトークンを見ますが、形態素解析器は六つの形態素を見ます。表層指標は、正しく活用された動詞と、有生性の一致に違反しているにもかかわらず尤もらしく見える幻覚とを区別できません。どちらも文字長が似た単一トークンです。

**自由語順。** Creeは語用論的に自由な語順を持ちます（Wolfart, 1973, §3.2）。*atim niwâpamâw* と *niwâpamâw atim*（「私は犬を見る」）はどちらも文法的に正しく、選択は語用論的なものであり、統語論的なものではありません。参照訳との語順の相違を減点する指標は、このようなペアすべてに対して偽陰性を生成します。

**形態論的等価性。** Creeには同一語の複数の有効な正書法表記があります（SROの変形、漸進的/母音長の交替）。*nikî-atoskân* と *nikî-atoskên* という翻訳は方言的に等価かもしれません。文字列照合指標は二つの異なる文字列を見ますが、言語学者は同一語を見ます。

**訓練データの欠如。** COMETのようなニューラル指標は、「良い」とはどういうことかを学習するために訓練データ（翻訳ペアに対する人間の品質判断）を必要とします。Creeについては、このデータが存在しません。COMETはスコアを生成できますが（多言語エンコーダにフォールバックします）、それらのスコアはCreeを話す人間の品質判断に対して検証されていません。根本的に異なる構造を持つ言語に適用された、ヨーロッパ言語パターンからの外挿にすぎません。

### 1.2 低リソース評価のパラドックス

これはパラドックスを生み出します：

> 機械翻訳を最も緊急に必要としている言語は、まさに最良の評価ツールが最も信頼できない言語です。

これらの言語の翻訳品質を測定できなければ、私たちは以下のことができません：
- 翻訳手法を客観的に比較する
- モデルが尤もらしく見えるナンセンスを幻覚しているときを検出する
- この分野が進歩しているかどうかを追跡する
- MT システム提供者に品質主張の説明責任を求める

結果として**連鎖的な失敗**が生じます：訓練データなし → エンコーダのカバレッジなし → 検証済み評価なし → 測定可能な進歩なし → 投資するインセンティブなし → 訓練データなし。

このサイクルを断ち切るには、私たちが持っていないリソース（訓練データ、大規模な人間の判断、ファインチューニングされたニューラルエンコーダ）に依存しない評価手法が必要です。私たちが*実際に*持っているリソースを活用する手法が必要です。

### 1.3 私たちが持っているもの

多くの低リソース言語について、数十年にわたる言語フィールドワークが、MT評価コミュニティがほぼ無視してきたツールとリソースを生み出しています：

| リソース | 提供するもの | カバレッジ |
|----------|-----------------|----------|
| **有限状態トランスデューサ（FST）** | 完全な形態素解析 — 言語内のすべての有効な語形 | GiellaLT、Apertium、NRC経由で約100以上の言語 |
| **対訳辞書** | 見出し語から語義へのマッピング | 数百の言語（Cree向けWolvengrey 2001：18,000以上のエントリ） |
| **形態素解析器** | 品詞タグ付け、見出し語化、活用パラダイム生成 | カバレッジが異なる数十の言語 |
| **記述文法** | 一致、語順、有生性、遠称を規定する規則 | ほとんどの記述済み言語で利用可能 |
| **言語学者の専門知識** | 正しい翻訳と誤った翻訳を識別できるコミュニティメンバー | すべての現存言語に定義上存在する |

これらのリソースは、計算言語学者、フィールド言語学者、言語コミュニティが数十年にわたって構築してきたものです。多くの場合、MT評価コミュニティとは無関係に。Plains CreeのFSTは、評価指標としてではなく言語記録ツールとして、アルバータ大学のAntti Arppe氏と同僚によって構築されました。UiTのGiellaLTインフラは、WMT共有タスクのためではなく、少数言語技術のために構築されました。

**Microevalは、これらの既存リソースを評価指標に変換する実践です。**

## 第2部：Microeval方法論

### 2.1 定義

**Microeval**は、言語固有の言語ツールとリソースを用いて、特定の言語に特化した機械翻訳評価指標を構築するための体系的な方法論です。Microevalスイートは：

1. 言語に依存しない指標では捉えられない**言語固有の知識を符号化する**
2. 新しい訓練データを必要とせず、**既存の言語インフラ**（FST、辞書、文法）を使用する
3. **決定論的で解釈可能なスコアを生成する** — すべてのスコアは特定の言語的判断に遡ることができる
4. エンジニアだけでなく**言語学者によって設計される** — 変形クラス、等価規則、検証ロジックは言語学的専門知識を反映する
5. 普遍的指標を置き換えるのではなく**補完する** — microevalはギャップを埋めるものであり、全空間を埋めるものではない

### 2.2 三層アーキテクチャ

完全なmicroevalスイートは、表層から意味論まで三つのレベルの分析で動作します：

| 層 | 答える問い | 使用ツール | LYSSコンポーネント |
|-------|------------------|-----------|----------------|
| **形態論的妥当性** | 「各語はこの言語の有効な形式か？」 | 有限状態トランスデューサ（FST） | LYSS-fst |
| **言語的等価性** | 「この翻訳は参照訳の許容可能な変形か？」 | 言語学者が精選した変形クラスを持つ決定論的リンター | LYSS-eq |
| **意味的忠実性** | 「この翻訳は原文の意味を保持しているか？」 | FST見出し語化 + 辞書語義 + 内容語の重複 | LYSS-sem |

これらの層は**代替ではなく累積的**です。翻訳が完全に正しいとみなされるには、三つすべてを通過する必要があります。幻覚された語は第1層で失敗します。参照訳とは異なるが正しい方言変形は第2層で捕捉されます。有効な語を有効な順序で使用しているが異なる意味を持つ翻訳は第3層で捕捉されます。

### 2.3 新しい言語のMicroevalスイートの構築方法

このセクションでは段階的なプロセスを説明します。Plains Cree（CRK）を実例として使用し、可能な限り一般化します。

#### ステップ1：利用可能なリソースの評価

何かを構築する前に、何が存在するかを調査します：

| リソース | 必要な用途 | 見つける方法 | 最低品質 |
|----------|-------------|----------------|-----------------|
| FST | 第1層（LYSS-fst） | GiellaLT、Apertium、NRCカタログを確認 | テストコーパス内の有効な語形の90%以上を受け入れること |
| 対訳辞書 | 第3層（LYSS-sem） | 言語記録プロジェクト、Wiktionary、コミュニティリソースを確認 | 見出し語から語義へのマッピングを持つ5,000以上のエントリ |
| 記述文法 | 第2層（LYSS-eq） | 出版された文法書、論文、コミュニティ著作の参考文献 | 核となる形態論的パラダイムを記述していること |
| バイリンガル話者 | 全層（検証） | コミュニティの連絡先、大学の言語プログラム | 検証実験のために最低3名の話者 |

**FSTが存在しない場合：** 第1層をスキップします。スイートは第2〜3層のみで動作するか、普遍的指標にフォールバックします（LYSSスコアリングのプロファイルB）。理想的ではありませんが、何もないよりはましです。

**辞書が存在しない場合：** 第3層をスキップするか、利用可能な語彙で縮小版を使用します。500エントリの辞書は18,000エントリのものほど有用ではありませんが、それでもシグナルを提供します。

#### ステップ2：形態論的妥当性ゲートの設定（LYSS-fst）

FSTが利用可能な場合：

1. 言語のアナライザバイナリ（GiellaLT向けHFST `.hfstol` 形式）を使用して**FSTをインストールする**
2. 代表的なコーパスで**カバレッジテストを実行する**：FSTはトークンの何パーセントを認識するか？
3. 予想されるFSTのギャップに対する**許可リストを構築する**：借用語、固有名詞、新語、略語
4. **ベースライン偽拒否率を計算する** — FSTが誤って拒否する有効な語の割合
5. **スコアリング閾値を設定する** — FST受け入れ率がどの値を下回ったら翻訳を形態論的に疑わしいとフラグを立てるか？

主要な指標は `fst_acceptance_rate` です：FSTが認識する出力語の割合。0.85という値は、語の85%が有効なCree形態論であることを意味します。15%は無効、借用語、またはFSTカバレッジのギャップです。

**重要な設計上の決定：** 偽拒否の問題。文語的なCreeで訓練されたFSTは、有効な口語形式を拒否します。パラダイムカバレッジが不完全なFSTは、有効だが稀な活用形を拒否します。許可リストはこれを軽減しますが、排除することはできません。*これがLYSS-fst単独では不十分な理由です* — 第2層と第3層と組み合わせる必要があります。

#### ステップ3：変形クラスの設計（LYSS-eq）

これは最も言語学的に要求の高いステップであり、自動化できません。対象言語の専門知識を持つ言語学者が以下を特定する必要があります：

**候補翻訳と参照翻訳の間のどのような違いが「許容可能」とみなされるべきか？**

Plains Creeについて、私たちは六つの変形クラスを特定しました：

| 変形クラス | 言語学的根拠 | 例 |
|--------------|-----------------|---------|
| `WORD_ORDER` | 語用論的に自由な語順（Wolfart 1973 §3.2） | *atim niwâpamâw* ≡ *niwâpamâw atim* |
| `ORTHOGRAPHIC` | SRO正書法変形、母音長交替 | *nikî-atoskân* ≡ *nikî-atoskên* |
| `OPTIONAL_PARTICLE` | 文法的に任意の談話小辞 | *êkwa nikî-atoskân* ≡ *nikî-atoskân* |
| `LEMMA_SYNONYM` | 辞書に記載された同義語 | *wâpamêw* ≡ *kanawâpamêw*（「見る」の意） |
| `PROGRESSIVE_AMBIGUITY` | 複数の有効な漸進形 | *ê-atoskêt* ≡ *atoskêw* |
| `INCLUSIVE_EXCLUSIVE` | 英語には存在しない一人称複数の区別 | *ki-wîcihânaw* ≡ *ni-wîcihânân* |

**これらのクラスは言語固有です。** 別の言語では異なるクラスが存在します。トルコ語には母音調和変形のクラスがあるかもしれません。日本語には敬語レジスターの交替があるかもしれません。イヌクティトゥット語には方言的な接尾辞変形があるかもしれません。

**設計プロセス：**
1. 100以上の翻訳ペア（原文 + 参照訳 + 候補訳）を収集する
2. 候補が参照訳と異なるが、バイリンガル話者が正しいとみなすすべてのケースを特定する
3. 違いを分類する — 複数のペアにわたって繰り返されるパターンを探す
4. 各パターンを決定論的な規則として形式化する（正規表現、集合演算、またはFSTトランスダクション）
5. 3名以上のバイリンガル話者で検証する：各変形クラスについて、それが許容可能であることに同意するか？
6. 反復する：一部のクラスは改良が必要になり、他のクラスは分割または統合が必要になる

#### ステップ4：意味的バリデータの構築（LYSS-sem）

意味的バリデータは「この翻訳は参照訳と同じ意味か？」という問いに答えます。四つの段階で動作します：

1. FSTを使用して**両方の翻訳を見出し語化する**（語根形式を抽出し、活用を除去する）
2. 対訳辞書を使用して**見出し語を語義にマッピングする**（Creeの見出し語 → 英語の語義）
3. **語義セットを比較する** — 候補の語義は参照訳の語義と重複するか？
4. **構造的制約を確認する** — 候補は既知の文法規則（有生性の一致、動詞形式、人称標示）に違反していないか？

バリデータは判定を生成します：`EXACT_MATCH`、`VALID`、`GRAMMAR_ISSUES`、`PARTIAL`、`INCOMPLETE`、`WRONG`、`NO_OUTPUT`。各判定は決定論的で追跡可能です。どの段階がフラグを立てたかを調べることで、翻訳が特定の判定を受けた*理由*を説明できます。

**最小限の実行可能バージョン：** FSTと辞書があれば、見出し語と語義の重複のみを行う簡略化された意味的バリデータ（第1〜3段階）を構築できます。第4段階（文法チェック）はより多くの言語工学を必要としますが、形態論的に複雑な言語に対して大きな価値を加えます。

#### ステップ5：評価ハーネスとの統合

Microevalスイートは、評価ハーネスにプラグインする指標プラグインのセットとしてパッケージ化されます：

1. 各指標は `MetricPlugin` プロトコルを実装します：`compute(entry) → dict`、`aggregate(results) → dict`
2. プラグイン検出システムは、対象言語コードに基づいて言語固有のプラグインを自動検出します
3. スコアは複合スコアリング関数に供給され、言語固有の重みプロファイルを使用してmicroeval指標と普遍的指標を組み合わせます

### 2.4 最小限の実行可能なMicroeval

すべての言語がすぐに三つの層すべてを必要とするわけではありません。各レベルでの最小限の有用な設定を以下に示します：

| 設定 | 必要なもの | 得られるもの | 構築期間 |
|--------------|--------------|-------------|---------------|
| **LYSS-fstのみ** | FST + 許可リスト | 形態論的妥当性ゲート — 幻覚された語形を捕捉 | 1〜2週間 |
| **LYSS-fst + LYSS-eq** | FST + 3〜6の変形クラス + 言語学者の時間 | 妥当性ゲート + 等価性検出 — 偽陰性を削減 | 4〜8週間 |
| **完全なLYSS** | FST + 変形クラス + 辞書 + 意味的バリデータ | 完全な言語固有評価 | 8〜16週間 |

推奨は、LYSS-fstから始めることです（高速、高インパクト、おそらく既に存在するFSTのみが必要）。そして段階的に層を追加します。

## 第3部：偽拒否の問題

### 3.1 偽拒否とは何か

すべてのmicroeval指標には偽拒否率があります：正しい翻訳が誤りとしてスコアリングされる確率です。

LYSS-fstでは、偽拒否は以下の場合に発生します：
- FSTが有効な語形をカバーしていない（不完全なパラダイム表）
- 翻訳にFSTが認識しない借用語が含まれている
- 翻訳に新語やブランド名が使用されている
- 翻訳にFSTの語彙にない方言形式が使用されている
- 翻訳に許可リストにない固有名詞が含まれている

LYSS-eqでは、偽拒否は以下の場合に発生します：
- 翻訳がどの変形クラスにもカバーされていない許容可能な変形を使用している
- 新しい変形クラスが必要だが、まだ特定されていない

LYSS-semでは、偽拒否は以下の場合に発生します：
- 見出し語が辞書にない
- 有効な翻訳が参照訳の見出し語セットにマッピングされない言い換え戦略を使用している

### 3.2 偽受け入れよりも重要な理由

評価において、偽拒否は偽受け入れよりも悪いです。偽拒否は*正しい*翻訳が*誤り*としてスコアリングされることを意味します。これは良い仕事をしている構築者を落胆させ、指標への信頼を損ないます。偽受け入れは*誤った*翻訳が*正しい*としてスコアリングされることを意味します。これは悪いことですが、複合スコアの他の指標によって捕捉されます。

偽拒否は複合します：LYSS-fstが語ごとに10%の偽拒否率を持つ場合、文が5語あると、少なくとも一語が偽拒否される確率は約41%です。つまり、ほぼ半数のすべての文で、翻訳が誤っているからではなく、FSTが不完全だからという理由で、FST受け入れ率が少なくとも一語分低下します。

### 3.3 軽減戦略

| 戦略 | メカニズム | 有効性 |
|----------|----------|---------------|
| **許可リスト** | 既知の借用語、固有名詞、略語をホワイトリストに登録 | 既知のギャップには高い、未知のギャップにはゼロ |
| **ファジーマッチング** | 既知の形式から編集距離1以内の語を受け入れる | タイポや軽微な正書法変形を捕捉 |
| **信頼スコアリング** | パラダイムの完全性メタデータによってFST結果を重み付け | パラダイムカバレッジメタデータが必要 |
| **カテゴリ固有の閾値** | ドメインごとに異なる閾値（医療は借用語が多い可能性） | ドメインタグ付きコーパスが必要 |
| **コミュニティが管理する許可リスト** | 話者がFSTが受け入れるべき語を提出 | 長期的に最も持続可能；コミュニティエンゲージメントインフラが必要 |

### 3.4 率の測定

偽拒否率は、代表的なコーパスで実証的に測定する必要があります：

1. 500以上の既知の有効なCree文のコーパスを取得する（教科書、レビュー済み翻訳）
2. すべての語をFSTで実行する
3. FSTが拒否するすべての語について、バイリンガル話者に分類してもらう：FSTが見逃した有効な語か、真に無効か？
4. `false_rejection_rate = valid_but_rejected / total_valid_words`

この測定は必須の検証実験の一つです（スコアリング仕様 §1.6）。

## 第4部：「正しい」とは誰が決めるのか？

### 4.1 評価の政治的側面

評価指標は中立的な道具ではありません。すべての指標は「正しい翻訳」とは何かという理論を埋め込んでおり、その理論には結果が伴います：

- 文語的なCreeから構築されたFSTは口語的なCreeを減点します。これはどのレジスターの言語が価値を持つかについての*政治的*選択です。
- ある方言形式を受け入れ、別の形式を受け入れない変形クラスは、暗黙的に言語を標準化します。標準化は、長い植民地主義の歴史を持つ政治的行為です。
- 正確な見出し語の重複を要求する意味的バリデータは、熟練した翻訳者が意図的に使用する重要な翻訳戦略である創造的な言い換えを減点します。

Microevalはこれらの選択を*明示的*にします。すべての変形クラス、すべての許可リストエントリ、すべての意味的等価規則は、個別の、文書化された、レビュー可能な決定です。これはバグではなく機能です：コミュニティが自分たちの言語がどのように評価されるかを規定する規則を検査し、異議を唱え、修正できることを意味します。

### 4.2 評価規則のコミュニティガバナンス

特に先住民言語については、評価の決定は外部の研究者やエンジニアではなく、言語コミュニティによって管理される必要があります。これは単に倫理的原則（そうではありますが）ではなく、正確性の要件です。変形が許容可能かどうかを判断できるのは流暢な話者だけです。

ガバナンスモデル：

1. **研究者が提案する** — 言語分析に基づいた変形クラス、許可リストエントリ、意味的規則
2. **話者がレビューする** — 各提案を承認、拒否、または修正する
3. **承認された規則は** 話者の帰属とともにコードベースにコミットされる
4. **異議のある規則は** コミュニティの議論のためにフラグが立てられる — 解決されるまでスコアリングから除外される
5. **コミュニティはいつでも** 承認済みセットから削除することで任意の規則を取り消すことができる

このモデルにはインフラ（評価ハーネス、バージョン管理、話者検証プロトコル）と関係性（研究者とコミュニティメンバー間の信頼）が必要です。このインフラの構築はmicroeval方法論の一部です。

### 4.3 方言変形

最も難しいガバナンスの問い：言語の二つの方言が形式について意見が分かれる場合、どちらが「正しい」のか？

Microevalの答え：**どちらも正しい。** 方言変形は方言タグを持つ追加の変形クラスとして表現されます。複合スコアは、評価が何を測定しようとしているかに応じて、方言ごとまたは方言をまたいで計算できます。

これには、コーパスが方言タグ付けされ、変形クラスが方言を認識していることが必要です。また、複数の方言の話者が検証に参加することも必要です。コーパスパートナーシップ戦略のドキュメントがこれらの要件を扱っています。

## 第5部：先行研究との関係

### 5.1 Microevalが主張しないこと

| 私たちが主張しないこと | 理由 |
|------------------------|---------|
| 「普遍的指標は役に立たない」 | 普遍的指標は不可欠なベースラインと言語横断的な比較可能性を提供します。Microevalは補完するものであり、置き換えるものではありません。 |
| 「ニューラル指標はLRLに機能しない」 | 機能します — 言語固有のデータでファインチューニングすれば。しかしそのデータはほとんど存在しません。Microevalは*今すぐ*機能します。 |
| 「FSTベースの評価は新しい」 | FSTはNLPで数十年使用されてきました。新しいのは、それらをMT評価指標として体系的に展開することです。 |
| 「LYSSはCOMETより優れている」 | わかりません — 人間との相関研究をまだ実施していません。多合成語言語に対してLYSSはより*情報量が多い*と考えていますが、証拠が得られるまでより*正確*だとは主張できません。 |

### 5.2 最も近い先行研究

| 研究 | Microevalとの関係 |
|------|--------------------------|
| **MorphEval**（Sánchez-Cartagena & Toral, 2024） | 形態論的現象のための対比テスト — 補完的。MorphEvalはシステムが形態論を*生成できるか*をテストします；LYSSは実際に*生成したか*をテストします。 |
| **CheckList**（Ribeiro et al., 2020） | NLPの行動テスト方法論 — 類似のパラダイム。CheckListは方法論；microevalも方法論であり、テストではなく評価に適用されます。 |
| **HyTER**（Dreyer & Marcu, 2012） | 意味的等価ラティス — LYSS-eqの概念的に最も近い先行研究。HyTERは言い換えを列挙します；LYSS-eqは形態論的変形を列挙します。HyTERは文ごとに手動のラティス構築が必要；LYSS-eqの規則はコーパス全体に適用されます。 |
| **Apertiumカバレッジ** | MT出力品質の代理としてFSTカバレッジを使用 — LYSS-fstを直接先取りしています。指標として形式化されておらず、スコアリングフレームワークに統合されていません。 |
| **FUSE**（AmericasNLP 2025） | 先住民アメリカ言語のための特徴ベース評価 — 精神的に最も類似しています。FUSEは言語的特徴を評価次元として提案します；LYSSは特定の言語に対して特定の特徴を実装します。直接比較が必要です。 |
| **AfriCOMET**（Wang & Adelani, 2024） | アフリカ言語向けにファインチューニングされたCOMET — ニューラル指標が*適応できる*ことを示しています。Microevalは、ファインチューニングデータが存在しない言語向けのルールベースの補完です。 |

### 5.3 主要な区別

形態論を考慮した評価に関するすべての先行研究は、以下のいずれかです：
1. 特定の言語に実装せずに**一般的なフレームワークを提案する**（FUSE、CheckList）
2. 訓練データが存在する**高リソース言語に実装する**（MorphEvalはヨーロッパのペアに焦点を当てる）
3. 私たちが持っていないデータを必要とする**ニューラル指標をファインチューニングする**（AfriCOMET）

Microevalは特に以下の場合のために設計されています：
- 言語ツール（FST、辞書）が存在する
- ニューラル指標のファインチューニング用訓練データが存在しない
- 言語の形態論的複雑さが表層指標を無効にする
- 評価はデータ収集キャンペーンの後ではなく、*今すぐ*運用可能でなければならない

## 第6部：未解決の問いと正直な限界

### 6.1 未解決の問い

1. **LYSS指標は人間の品質判断と相関するか？** わかりません。必要な人間との相関研究（200以上の文ペア、3名以上のバイリンガル話者）はまだ実施されていません。それが実施されるまで、LYSSスコアはエンジニアリングの推定値であり、検証済みの品質測定値ではありません。

2. **言語が変化するにつれてLYSS指標はどのように振る舞うか？** 現存言語は進化します — 新しい借用語、変化する方言、新興の新語。FSTと変形クラスは維持される必要があります。メンテナンスの負担はどの程度か？わかりません。

3. **有用なLYSS-fstのための最低限のFST品質は何か？** FSTが語彙の60%しかカバーしていない場合、LYSS-fstはまだ有用か、それともノイズがシグナルを圧倒するか？実証的な証拠が必要です。

4. **Microevalは非形態論的な課題に機能するか？** 声調の区別、クリック子音、または表語文字を持つ言語は、FSTが対処しない評価上の課題を提示します。Microevalは適用できないかもしれません — または異なるツールが必要かもしれません。

5. **コールドスタート問題をどのように処理するか？** Microevalスイートの構築には言語学的専門知識が必要です。活発な計算言語学コミュニティを持たない言語については、誰がその作業を行うのか？

### 6.2 LYSSの正直な限界

| 限界 | 深刻度 | 軽減策 |
|-----------|----------|-----------|
| 人間との相関データなし | 🔴 重大 | 必須検証実験 #1 |
| FST偽拒否率が未測定 | 🔴 重大 | 必須検証実験 #2 |
| 一言語（CRK）のみに実装 | 🟡 重要 | 第二言語ポート（Northern Sámi）を計画中 |
| 変形クラスが不完全な可能性 | 🟡 重要 | コミュニティレビュー + 継続的な追加 |
| 意味的バリデータがspaCyを必要とする | 🟡 重要 | オプションの依存関係；グレースフルデグラデーション |
| 辞書カバレッジがLYSS-sem品質に影響 | 🟡 重要 | 最低辞書サイズ要件を文書化 |
| 流暢さや自然さを検出できない | 🟡 重要 | 人間による評価またはニューラル指標が必要 |
| 拡張に言語学的専門知識が必要 | 🟡 重要 | 方法論の文書化（本稿）が障壁を低減 |

### 6.3 今後の道筋

> *「一般化できるものだけに焦点を当てると、それが機能しない場所を必然的に忘れてしまいます — そしてこれらの言語と、そのすべての知識と知恵を失うことになります。」*

Microevalは評価問題の解決策ではありません。それは実践です — 各言語を異なるものにする要素に注意を払い、その注意を動作するコードに符号化するという規律です。この実践は労力を要し、言語固有であり、決して完成しません。しかし、普遍的指標のパラダイムが生み出せないものを生み出します：評価する言語で語る評価です。
