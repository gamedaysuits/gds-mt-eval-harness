# Microeval: เมตริกการประเมินผลเฉพาะภาษาสำหรับการแปลด้วยเครื่อง

***วิธีการสร้างเมตริกการประเมินผลที่ปรับให้เหมาะกับแต่ละภาษาโดยใช้ FST พจนานุกรม และกฎความเท่าเทียมที่ภาษาศาสตร์ผู้เชี่ยวชาญกำหนด — และเหตุใดสาขานี้จึงต้องการสิ่งนี้***

---

> *"ขอบเขตของภาษาของฉัน คือขอบเขตของโลกของฉัน"*
> — Ludwig Wittgenstein, *Tractatus Logico-Philosophicus* (1921)

---

## บทนำ

ชุมชนการประเมินผลการแปลด้วยเครื่องใช้เวลากว่าสองทศวรรษในการค้นหาเมตริกสากล — มาตรวัดคุณภาพการแปลที่ใช้ได้กับทุกภาษา ทุกโดเมน และทุกประเภทภาษา การค้นหานี้ได้ผลิตเครื่องมือที่โดดเด่น ได้แก่ BLEU (Papineni et al., 2002), chrF++ (Popović, 2017), COMET (Rei et al., 2020), MetricX (Juraska et al., 2023) สำหรับภาษาประมาณ 20 ภาษาที่ครองงาน WMT shared tasks เครื่องมือเหล่านี้ทำงานได้ดี

แต่สำหรับภาษาอื่นอีกกว่า 7,000 ภาษา เครื่องมือเหล่านี้ไม่ได้ผล

บทความนี้โต้แย้งว่า **การค้นหาเมตริกสากล เมื่อนำไปใช้กับภาษาที่มีทรัพยากรน้อยและภาษาที่มีความซับซ้อนทางสัณฐานวิทยา ไม่ใช่แค่ไม่สมบูรณ์ — แต่เป็นกระบวนทัศน์ที่ผิด** เราเสนอ **microeval**: วิธีการสร้างเมตริกการประเมินผลที่ปรับให้เหมาะกับแต่ละภาษาโดยใช้เครื่องมือทางภาษาศาสตร์ที่ดีที่สุดที่มีอยู่ ได้แก่ finite-state transducers พจนานุกรมสองภาษา ตัววิเคราะห์สัณฐานวิทยา และกฎความเท่าเทียมที่ภาษาศาสตร์ผู้เชี่ยวชาญกำหนด

Microeval ไม่ใช่เมตริก แต่เป็น *แนวปฏิบัติ* — กระบวนการที่เป็นระบบสำหรับสร้างโครงสร้างพื้นฐานการประเมินผลที่เข้ารหัสความรู้เฉพาะภาษา แนวปฏิบัตินี้ผลิตเมตริก ซึ่งเรารวบรวมไว้ภายใต้ชื่อกรอบงาน **LYSS** (Linguistically-informed Yield & Structural Scoring) แต่สิ่งที่เราเสนอคือวิธีการ ไม่ใช่เมตริกใดเมตริกหนึ่งที่ผลิตออกมา

เอกสารนี้เป็นเอกสารประกอบของ:
- [Measuring the Immeasurable](/docs/context/mt-evaluation-landscape) — การสำรวจภูมิทัศน์การประเมินผล ซึ่งวาง LYSS ไว้ท่ามกลางเมตริกที่มีอยู่
- [The Scoring Specification](/docs/specifications/scoring) — ข้อกำหนดทางเทคนิคสำหรับนิยามเมตริก น้ำหนัก และการให้คะแนนแบบผสม
- [The Corpus Partnership Strategy](/docs/specifications/corpus-partnership) — ขั้นตอนการทำงานจริงสำหรับการสร้างคลังข้อมูลการประเมินผล

เอกสารเหล่านั้นอธิบาย *ว่า* LYSS คืออะไรและ *อยู่ที่ไหน* ในภาพรวม เอกสารนี้ตอบคำถามที่ลึกกว่า: *ทำไม* การประเมินผลเฉพาะภาษาจึงจำเป็น? *อย่างไร* จึงจะสร้างมันสำหรับภาษาใหม่? และ *ใคร* เป็นผู้ตัดสินว่าอะไรคือ "ถูกต้อง"?

## ส่วนที่ 1: เหตุใดเมตริกสากลจึงล้มเหลวกับภาษาที่มีทรัพยากรน้อย

### 1.1 สมมติฐานความเป็นสากล

เมตริกการประเมินผล MT ทุกตัวที่สำคัญนับตั้งแต่ BLEU ล้วนอาศัยสมมติฐานโดยนัย: ว่า *กลไก* ของคุณภาพการแปลนั้นเป็นอิสระจากภาษา แม้ว่า *พารามิเตอร์* จะแตกต่างกัน BLEU นับการทับซ้อนของ n-gram chrF++ นับการทับซ้อนของ character n-gram COMET ฝึกโมเดล regression บนการตัดสินของมนุษย์ ทั้งหมดสมมติว่าโครงสร้างสัญญาณ — สิ่งที่ทำให้การแปล "ดี" — สามารถจับได้ด้วยอัลกอริทึมที่ไม่ขึ้นกับภาษา ซึ่งอาจปรับแต่งด้วยข้อมูลเฉพาะภาษา

สำหรับคู่ภาษายุโรปที่มีทรัพยากรสูง สมมติฐานนี้ใช้ได้ดีพอ งาน WMT metrics shared tasks แสดงให้เห็นความสัมพันธ์สูงกับมนุษย์สำหรับ English↔German, English↔Czech, English↔Chinese เมตริกอาจไม่เห็นด้วยในกรณีขอบ แต่เห็นด้วยกับการกระจายของคุณภาพ

สำหรับภาษาโพลีซินเทติก เช่น Plains Cree (nêhiyawêwin) สมมติฐานนี้พังทลายในหลายระดับ:

**ความทึบแสงทางสัณฐานวิทยา** กริยาเดียวในภาษา Cree สามารถบรรจุข้อมูลได้มากเท่ากับประโยคภาษาอังกฤษทั้งประโยค คำว่า *nikî-wîcihâw* ("ฉันช่วยเขา/เธอ") เข้ารหัสบุคคล จำนวน ความมีชีวิต ทิศทาง และกาลในรูปแบบผันเดียว เมตริก n-gram เห็นหนึ่ง token; ตัววิเคราะห์สัณฐานวิทยาเห็นหกหน่วยคำ เมตริกพื้นผิวไม่สามารถแยกแยะระหว่างกริยาที่ผันถูกต้องกับการสร้างคำที่ดูสมเหตุสมผลแต่ละเมิดข้อตกลงความมีชีวิต — ทั้งสองเป็น token เดียวที่มีความยาวอักขระใกล้เคียงกัน

**ลำดับคำอิสระ** ภาษา Cree มีลำดับคำที่อิสระตามหลักปฏิบัติ (Wolfart, 1973, §3.2) ประโยค *atim niwâpamâw* และ *niwâpamâw atim* ("ฉันเห็นสุนัข") ถูกต้องทางไวยากรณ์ทั้งคู่ — การเลือกเป็นเรื่องของหลักปฏิบัติ ไม่ใช่วากยสัมพันธ์ เมตริกใดที่ลงโทษการเบี่ยงเบนลำดับคำจากการแปลอ้างอิงจะสร้างผลลบเท็จสำหรับทุกคู่ดังกล่าว

**ความเท่าเทียมทางสัณฐานวิทยา** ภาษา Cree มีการแทนค่าอักขรวิธีที่ถูกต้องหลายแบบสำหรับคำเดียวกัน (รูปแบบ SRO, การสลับความยาวสระ/เสียงสระ) การแปล *nikî-atoskân* และ *nikî-atoskên* อาจเท่าเทียมกันในเชิงสำเนียง เมตริกการจับคู่สตริงเห็นสองสตริงที่แตกต่างกัน; นักภาษาศาสตร์เห็นคำเดียวกัน

**การขาดข้อมูลฝึก** เมตริกประสาทเช่น COMET ต้องการข้อมูลฝึก — การตัดสินคุณภาพของมนุษย์บนคู่การแปล — เพื่อเรียนรู้ว่า "ดี" หมายความว่าอะไร สำหรับภาษา Cree ข้อมูลนี้ไม่มีอยู่ COMET ยังคงสร้างคะแนนได้ (ใช้ตัวเข้ารหัสหลายภาษาสำรอง) แต่คะแนนเหล่านั้นยังไม่ได้รับการตรวจสอบกับการตัดสินคุณภาพของผู้พูดภาษา Cree คนใด คะแนนเหล่านั้นเป็นการประมาณจากรูปแบบภาษายุโรป ที่นำไปใช้กับภาษาที่มีโครงสร้างแตกต่างกันโดยพื้นฐาน

### 1.2 ความขัดแย้งของการประเมินผลภาษาที่มีทรัพยากรน้อย

สิ่งนี้สร้างความขัดแย้ง:

> ภาษาที่ต้องการการแปลด้วยเครื่องมากที่สุดอย่างเร่งด่วน คือภาษาที่เครื่องมือประเมินผลที่ดีที่สุดมีความน่าเชื่อถือน้อยที่สุด

หากเราไม่สามารถวัดคุณภาพการแปลสำหรับภาษาเหล่านี้ เราไม่สามารถ:
- เปรียบเทียบวิธีการแปลอย่างเป็นกลาง
- ตรวจจับเมื่อโมเดลสร้างข้อความที่ดูสมเหตุสมผลแต่ไม่มีความหมาย
- ติดตามว่าสาขานี้กำลังก้าวหน้าหรือไม่
- ตรวจสอบผู้ให้บริการระบบ MT สำหรับการอ้างสิทธิ์คุณภาพ

ผลลัพธ์คือ **ความล้มเหลวแบบต่อเนื่อง**: ไม่มีข้อมูลฝึก → ไม่มีการครอบคลุมตัวเข้ารหัส → ไม่มีการประเมินผลที่ตรวจสอบแล้ว → ไม่มีความก้าวหน้าที่วัดได้ → ไม่มีแรงจูงใจในการลงทุน → ไม่มีข้อมูลฝึก

การทำลายวงจรนี้ต้องการวิธีการประเมินผลที่ไม่ขึ้นอยู่กับทรัพยากรที่เราไม่มี (ข้อมูลฝึก การตัดสินของมนุษย์ในระดับใหญ่ ตัวเข้ารหัสประสาทที่ปรับแต่งแล้ว) แต่ต้องการวิธีการที่ใช้ประโยชน์จากทรัพยากรที่เรา *มี*

### 1.3 สิ่งที่เรามี

สำหรับภาษาที่มีทรัพยากรน้อยหลายภาษา งานภาคสนามทางภาษาศาสตร์หลายทศวรรษได้ผลิตเครื่องมือและทรัพยากรที่ชุมชนการประเมินผล MT ส่วนใหญ่ยังไม่ได้นำมาใช้:

| ทรัพยากร | สิ่งที่ให้ | การครอบคลุม |
|----------|-----------------|----------|
| **Finite-State Transducers (FSTs)** | การวิเคราะห์สัณฐานวิทยาสมบูรณ์ — รูปแบบคำที่ถูกต้องทุกรูปแบบในภาษา | ~100+ ภาษาผ่าน GiellaLT, Apertium, NRC |
| **พจนานุกรมสองภาษา** | การแมปจาก lemma ไปยัง gloss | หลายร้อยภาษา (Wolvengrey 2001 สำหรับ Cree: 18,000+ รายการ) |
| **ตัววิเคราะห์สัณฐานวิทยา** | การแท็กส่วนของคำพูด การทำ lemmatization การสร้างกระบวนทัศน์การผัน | หลายสิบภาษาที่มีการครอบคลุมแตกต่างกัน |
| **ไวยากรณ์เชิงพรรณนา** | กฎที่ควบคุมข้อตกลง ลำดับคำ ความมีชีวิต การอ้างอิงทางอ้อม | มีสำหรับภาษาที่มีเอกสารส่วนใหญ่ |
| **ความเชี่ยวชาญของนักภาษาศาสตร์** | สมาชิกชุมชนที่สามารถระบุการแปลที่ถูกต้องและไม่ถูกต้อง | มีอยู่โดยนิยามสำหรับทุกภาษาที่ยังมีชีวิต |

ทรัพยากรเหล่านี้สร้างขึ้นโดยนักภาษาศาสตร์เชิงคำนวณ นักภาษาศาสตร์ภาคสนาม และชุมชนภาษาในช่วงหลายทศวรรษ — มักไม่มีความเชื่อมโยงกับชุมชนการประเมินผล MT FST สำหรับ Plains Cree สร้างขึ้นที่มหาวิทยาลัย Alberta โดย Antti Arppe และเพื่อนร่วมงานในฐานะเครื่องมือบันทึกภาษา ไม่ใช่เมตริกการประเมินผล โครงสร้างพื้นฐาน GiellaLT ที่ UiT สร้างขึ้นสำหรับเทคโนโลยีภาษาชนกลุ่มน้อย ไม่ใช่สำหรับ WMT shared tasks

**Microeval คือแนวปฏิบัติในการเปลี่ยนทรัพยากรที่มีอยู่เหล่านี้ให้เป็นเมตริกการประเมินผล**

## ส่วนที่ 2: วิธีการ Microeval

### 2.1 นิยาม

**Microeval** คือวิธีการที่เป็นระบบสำหรับสร้างเมตริกการประเมินผลการแปลด้วยเครื่องที่ปรับให้เหมาะกับภาษาเฉพาะ โดยใช้เครื่องมือและทรัพยากรทางภาษาศาสตร์เฉพาะภาษา ชุด microeval:

1. **เข้ารหัสความรู้เฉพาะภาษา** ที่ไม่สามารถจับได้ด้วยเมตริกที่ไม่ขึ้นกับภาษา
2. **ใช้โครงสร้างพื้นฐานทางภาษาศาสตร์ที่มีอยู่** (FSTs, พจนานุกรม, ไวยากรณ์) แทนที่จะต้องการข้อมูลฝึกใหม่
3. **ผลิตคะแนนที่กำหนดได้และตีความได้** — ทุกคะแนนสามารถสืบย้อนไปยังการตัดสินทางภาษาศาสตร์เฉพาะ
4. **ออกแบบโดยนักภาษาศาสตร์** ไม่ใช่แค่วิศวกร — คลาสตัวแปร กฎความเท่าเทียม และตรรกะการตรวจสอบสะท้อนความเชี่ยวชาญทางภาษาศาสตร์
5. **เสริมแทนที่จะแทนที่** เมตริกสากล — microeval เติมเต็มช่องว่าง ไม่ใช่ทั้งหมด

### 2.2 สถาปัตยกรรมสามชั้น

ชุด microeval ที่สมบูรณ์ทำงานในสามระดับของการวิเคราะห์ จากพื้นผิวไปยังความหมาย:

| ชั้น | คำถามที่ตอบ | เครื่องมือที่ใช้ | องค์ประกอบ LYSS |
|-------|------------------|-----------|----------------|
| **ความถูกต้องทางสัณฐานวิทยา** | "แต่ละคำเป็นรูปแบบที่ถูกต้องในภาษานี้หรือไม่?" | Finite-state transducer (FST) | LYSS-fst |
| **ความเท่าเทียมทางภาษาศาสตร์** | "การแปลนี้เป็นตัวแปรที่ยอมรับได้ของการอ้างอิงหรือไม่?" | Linter แบบกำหนดได้พร้อมคลาสตัวแปรที่นักภาษาศาสตร์กำหนด | LYSS-eq |
| **ความซื่อสัตย์ทางความหมาย** | "การแปลนี้รักษาความหมายของต้นฉบับหรือไม่?" | การทำ lemmatization ด้วย FST + gloss จากพจนานุกรม + การทับซ้อนของคำเนื้อหา | LYSS-sem |

ชั้นเหล่านี้ **สะสม ไม่ใช่ทางเลือก** การแปลต้องผ่านทั้งสามชั้นจึงจะถือว่าถูกต้องสมบูรณ์ คำที่สร้างขึ้นเองล้มเหลวที่ชั้น 1 ตัวแปรสำเนียงที่ถูกต้องแต่แตกต่างจากการอ้างอิงถูกจับที่ชั้น 2 การแปลที่ใช้คำที่ถูกต้องในลำดับที่ถูกต้องแต่มีความหมายต่างออกไปถูกจับที่ชั้น 3

### 2.3 วิธีสร้างชุด Microeval สำหรับภาษาใหม่

ส่วนนี้อธิบายกระบวนการทีละขั้นตอน เราใช้ Plains Cree (CRK) เป็นตัวอย่างที่ใช้งานได้จริงและสรุปทั่วไปเมื่อเป็นไปได้

#### ขั้นตอนที่ 1: ประเมินทรัพยากรที่มีอยู่

ก่อนสร้างสิ่งใด ให้ทำรายการสิ่งที่มีอยู่:

| ทรัพยากร | จำเป็นสำหรับ | วิธีค้นหา | คุณภาพขั้นต่ำ |
|----------|-------------|----------------|-----------------|
| FST | ชั้น 1 (LYSS-fst) | ตรวจสอบแคตาล็อก GiellaLT, Apertium, NRC | ต้องยอมรับ >90% ของรูปแบบคำที่ถูกต้องในคลังทดสอบ |
| พจนานุกรมสองภาษา | ชั้น 3 (LYSS-sem) | ตรวจสอบโครงการบันทึกภาษา, Wiktionary, ทรัพยากรชุมชน | >5,000 รายการพร้อมการแมป lemma-to-gloss |
| ไวยากรณ์เชิงพรรณนา | ชั้น 2 (LYSS-eq) | ไวยากรณ์ที่ตีพิมพ์, วิทยานิพนธ์, เอกสารอ้างอิงที่ชุมชนเขียน | ต้องบันทึกกระบวนทัศน์สัณฐานวิทยาหลัก |
| ผู้พูดสองภาษา | ทุกชั้น (การตรวจสอบ) | ผู้ติดต่อในชุมชน, โปรแกรมภาษาของมหาวิทยาลัย | ผู้พูดขั้นต่ำ 3 คนสำหรับการทดลองตรวจสอบ |

**หากไม่มี FST:** ข้ามชั้น 1 ชุดทำงานบนชั้น 2–3 เท่านั้น หรือใช้เมตริกสากลแทน (Profile B ในการให้คะแนน LYSS) ไม่ใช่สิ่งที่ดีที่สุด แต่ดีกว่าไม่มีอะไร

**หากไม่มีพจนานุกรม:** ข้ามชั้น 3 หรือใช้เวอร์ชันที่ลดลงพร้อมคำศัพท์ที่มีอยู่ พจนานุกรม 500 รายการมีประโยชน์น้อยกว่าพจนานุกรม 18,000 รายการ แต่ยังคงให้สัญญาณ

#### ขั้นตอนที่ 2: กำหนดค่าประตูความถูกต้องทางสัณฐานวิทยา (LYSS-fst)

หากมี FST:

1. **ติดตั้ง FST** โดยใช้ไบนารีตัววิเคราะห์ของภาษา (รูปแบบ HFST `.hfstol` สำหรับ GiellaLT)
2. **ทดสอบการครอบคลุม** บนคลังข้อมูลตัวแทน: FST รู้จักโทเค็นกี่เปอร์เซ็นต์?
3. **สร้าง allowlist** สำหรับช่องว่าง FST ที่คาดหวัง: คำยืม ชื่อเฉพาะ คำใหม่ คำย่อ
4. **คำนวณอัตราการปฏิเสธเท็จพื้นฐาน** — เปอร์เซ็นต์ของคำที่ถูกต้องที่ FST ปฏิเสธอย่างไม่ถูกต้อง
5. **กำหนดเกณฑ์การให้คะแนน** — ต่ำกว่าอัตราการยอมรับ FST เท่าใดที่เราจะตั้งค่าสถานะการแปลว่าน่าสงสัยทางสัณฐานวิทยา?

เมตริกหลักคือ `fst_acceptance_rate`: สัดส่วนของคำในผลลัพธ์ที่ FST รู้จัก อัตรา 0.85 หมายความว่า 85% ของคำเป็นสัณฐานวิทยา Cree ที่ถูกต้อง; 15% เป็นคำที่ไม่ถูกต้อง คำยืม หรือช่องว่างการครอบคลุม FST

**การตัดสินใจออกแบบที่สำคัญ:** ปัญหาการปฏิเสธเท็จ FST ที่ฝึกบนภาษา Cree วรรณกรรมอย่างเป็นทางการจะปฏิเสธรูปแบบภาษาพูดที่ถูกต้อง FST ที่มีการครอบคลุมกระบวนทัศน์ไม่สมบูรณ์จะปฏิเสธการผันที่ถูกต้องแต่หายาก allowlist ช่วยบรรเทาปัญหานี้ แต่ไม่สามารถขจัดได้ *นี่คือเหตุผลที่ LYSS-fst เพียงอย่างเดียวไม่เพียงพอ* — ต้องรวมกับชั้น 2 และ 3

#### ขั้นตอนที่ 3: ออกแบบคลาสตัวแปร (LYSS-eq)

นี่คือขั้นตอนที่ต้องการความเชี่ยวชาญทางภาษาศาสตร์มากที่สุด และไม่สามารถทำให้เป็นอัตโนมัติได้ นักภาษาศาสตร์ที่มีความเชี่ยวชาญในภาษาเป้าหมายต้องระบุ:

**ความแตกต่างประเภทใดระหว่างการแปลผู้สมัครและการแปลอ้างอิงที่ควรถือว่า "ยอมรับได้"?**

สำหรับ Plains Cree เราระบุคลาสตัวแปรหกคลาส:

| คลาสตัวแปร | พื้นฐานทางภาษาศาสตร์ | ตัวอย่าง |
|--------------|-----------------|---------|
| `WORD_ORDER` | ลำดับคำที่อิสระตามหลักปฏิบัติ (Wolfart 1973 §3.2) | *atim niwâpamâw* ≡ *niwâpamâw atim* |
| `ORTHOGRAPHIC` | รูปแบบการสะกด SRO, การสลับความยาวสระ | *nikî-atoskân* ≡ *nikî-atoskên* |
| `OPTIONAL_PARTICLE` | อนุภาควาทกรรมที่เป็นทางเลือกทางไวยากรณ์ | *êkwa nikî-atoskân* ≡ *nikî-atoskân* |
| `LEMMA_SYNONYM` | คำพ้องความหมายที่มีในพจนานุกรม | *wâpamêw* ≡ *kanawâpamêw* (สำหรับ "sees") |
| `PROGRESSIVE_AMBIGUITY` | รูปแบบ progressive ที่ถูกต้องหลายรูปแบบ | *ê-atoskêt* ≡ *atoskêw* |
| `INCLUSIVE_EXCLUSIVE` | ความแตกต่างบุรุษที่หนึ่งพหูพจน์ที่ไม่มีในภาษาอังกฤษ | *ki-wîcihânaw* ≡ *ni-wîcihânân* |

**คลาสเหล่านี้เฉพาะเจาะจงกับภาษา** ภาษาอื่นจะมีคลาสที่แตกต่างกัน — ภาษาตุรกีอาจมีคลาสสำหรับตัวแปรสัมพันธ์เสียงสระ ภาษาญี่ปุ่นสำหรับการสลับระดับความสุภาพ ภาษา Inuktitut สำหรับการแปรปรวนของคำต่อท้ายตามสำเนียง

**กระบวนการออกแบบ:**
1. รวบรวมคู่การแปล 100+ คู่ (ต้นฉบับ + อ้างอิง + ผู้สมัคร)
2. ระบุทุกกรณีที่ผู้สมัครแตกต่างจากการอ้างอิงแต่ผู้พูดสองภาษาจะถือว่าถูกต้อง
3. จัดหมวดหมู่ความแตกต่าง — มองหารูปแบบที่เกิดซ้ำในหลายคู่
4. ทำให้แต่ละรูปแบบเป็นกฎที่กำหนดได้ (regex, การดำเนินการเซต, หรือ FST transduction)
5. ตรวจสอบกับผู้พูดสองภาษา 3+ คน: สำหรับแต่ละคลาสตัวแปร พวกเขาเห็นด้วยว่ายอมรับได้หรือไม่?
6. ทำซ้ำ: บางคลาสจะต้องปรับแต่ง บางคลาสจะต้องแยกหรือรวม

#### ขั้นตอนที่ 4: สร้างตัวตรวจสอบความหมาย (LYSS-sem)

ตัวตรวจสอบความหมายตอบคำถาม: "การแปลนี้มีความหมายเหมือนกับการอ้างอิงหรือไม่?" ทำงานในสี่ขั้นตอน:

1. **ทำ lemmatize ทั้งสองการแปล** โดยใช้ FST (แยกรูปแบบราก ลบการผัน)
2. **แมป lemma ไปยัง gloss** โดยใช้พจนานุกรมสองภาษา (Cree lemma → gloss ภาษาอังกฤษ)
3. **เปรียบเทียบชุด gloss** — gloss ของผู้สมัครทับซ้อนกับ gloss ของการอ้างอิงหรือไม่?
4. **ตรวจสอบข้อจำกัดโครงสร้าง** — ผู้สมัครละเมิดกฎไวยากรณ์ที่ทราบ (ข้อตกลงความมีชีวิต รูปแบบกริยา การทำเครื่องหมายบุคคล) หรือไม่?

ตัวตรวจสอบผลิตคำตัดสิน: `EXACT_MATCH`, `VALID`, `GRAMMAR_ISSUES`, `PARTIAL`, `INCOMPLETE`, `WRONG`, `NO_OUTPUT` แต่ละคำตัดสินกำหนดได้และสืบย้อนได้ — คุณสามารถอธิบาย *ว่าทำไม* การแปลได้รับคำตัดสินที่กำหนดโดยตรวจสอบว่าขั้นตอนใดตั้งค่าสถานะ

**เวอร์ชันที่ใช้งานได้ขั้นต่ำ:** หากคุณมี FST และพจนานุกรม คุณสามารถสร้างตัวตรวจสอบความหมายแบบง่ายที่ทำเฉพาะการทับซ้อน lemma-gloss (ขั้นตอน 1–3) ขั้นตอน 4 (การตรวจสอบไวยากรณ์) ต้องการวิศวกรรมภาษาศาสตร์เพิ่มเติมแต่เพิ่มคุณค่าอย่างมีนัยสำคัญสำหรับภาษาที่มีความซับซ้อนทางสัณฐานวิทยา

#### ขั้นตอนที่ 5: รวมเข้ากับ Evaluation Harness

ชุด microeval ถูกบรรจุเป็นชุดปลั๊กอินเมตริกที่เชื่อมต่อกับ evaluation harness:

1. แต่ละเมตริกใช้โปรโตคอล `MetricPlugin`: `compute(entry) → dict`, `aggregate(results) → dict`
2. ระบบค้นพบปลั๊กอินตรวจจับปลั๊กอินเฉพาะภาษาโดยอัตโนมัติตามรหัสภาษาเป้าหมาย
3. คะแนนถูกส่งไปยังฟังก์ชันการให้คะแนนแบบผสม ซึ่งรวมเมตริก microeval กับเมตริกสากลโดยใช้โปรไฟล์น้ำหนักเฉพาะภาษา

### 2.4 Microeval ที่ใช้งานได้ขั้นต่ำ

ไม่ใช่ทุกภาษาที่ต้องการทั้งสามชั้นในทันที นี่คือการกำหนดค่าที่มีประโยชน์ขั้นต่ำในแต่ละระดับ:

| การกำหนดค่า | สิ่งที่คุณต้องการ | สิ่งที่คุณได้รับ | เวลาในการสร้าง |
|--------------|--------------|-------------|---------------|
| **LYSS-fst เท่านั้น** | FST + allowlist | ประตูความถูกต้องทางสัณฐานวิทยา — จับรูปแบบคำที่สร้างขึ้นเอง | 1–2 สัปดาห์ |
| **LYSS-fst + LYSS-eq** | FST + คลาสตัวแปร 3–6 คลาส + เวลาของนักภาษาศาสตร์ | ประตูความถูกต้อง + การตรวจจับความเท่าเทียม — ลดผลลบเท็จ | 4–8 สัปดาห์ |
| **LYSS เต็มรูปแบบ** | FST + คลาสตัวแปร + พจนานุกรม + ตัวตรวจสอบความหมาย | การประเมินผลเฉพาะภาษาสมบูรณ์ | 8–16 สัปดาห์ |

คำแนะนำคือเริ่มต้นด้วย LYSS-fst (รวดเร็ว ผลกระทบสูง ต้องการเฉพาะ FST ที่น่าจะมีอยู่แล้ว) และเพิ่มชั้นทีละน้อย

## ส่วนที่ 3: ปัญหาการปฏิเสธเท็จ

### 3.1 คืออะไร

เมตริก microeval ทุกตัวมีอัตราการปฏิเสธเท็จ: ความน่าจะเป็นที่การแปลที่ถูกต้องจะถูกให้คะแนนว่าไม่ถูกต้อง

สำหรับ LYSS-fst การปฏิเสธเท็จเกิดขึ้นเมื่อ:
- FST ไม่ครอบคลุมรูปแบบคำที่ถูกต้อง (ตารางกระบวนทัศน์ไม่สมบูรณ์)
- การแปลมีคำยืมที่ FST ไม่รู้จัก
- การแปลใช้คำใหม่หรือชื่อแบรนด์
- การแปลใช้รูปแบบสำเนียงที่ไม่อยู่ในพจนานุกรม FST
- การแปลมีชื่อเฉพาะที่ไม่อยู่ใน allowlist

สำหรับ LYSS-eq การปฏิเสธเท็จเกิดขึ้นเมื่อ:
- การแปลใช้ตัวแปรที่ยอมรับได้ซึ่งไม่ครอบคลุมโดยคลาสตัวแปรใด
- จำเป็นต้องมีคลาสตัวแปรใหม่แต่ยังไม่ได้ระบุ

สำหรับ LYSS-sem การปฏิเสธเท็จเกิดขึ้นเมื่อ:
- lemma ไม่อยู่ในพจนานุกรม
- การแปลที่ถูกต้องใช้กลยุทธ์การถอดความที่ไม่แมปกับชุด lemma ของการอ้างอิง

### 3.2 เหตุใดจึงสำคัญกว่าการยอมรับเท็จ

ในการประเมินผล การปฏิเสธเท็จแย่กว่าการยอมรับเท็จ การปฏิเสธเท็จหมายความว่าการแปลที่ *ถูกต้อง* ถูกให้คะแนนว่า *ผิด* — สิ่งนี้ทำให้ผู้สร้างที่ทำงานดีท้อแท้ และบ่อนทำลายความไว้วางใจในเมตริก การยอมรับเท็จหมายความว่าการแปลที่ *ผิด* ถูกให้คะแนนว่า *ถูกต้อง* — สิ่งนี้ไม่ดี แต่ถูกจับโดยเมตริกอื่นในแบบผสม

การปฏิเสธเท็จสะสม: หาก LYSS-fst มีอัตราการปฏิเสธเท็จ 10% ต่อคำ และประโยคมี 5 คำ ความน่าจะเป็นที่คำอย่างน้อยหนึ่งคำจะถูกปฏิเสธเท็จคือ ~41% ซึ่งหมายความว่าเกือบครึ่งหนึ่งของประโยคทั้งหมดจะมีอัตราการยอมรับ FST ลดลงอย่างน้อยหนึ่งคำ — ไม่ใช่เพราะการแปลผิด แต่เพราะ FST ไม่สมบูรณ์

|----------|----------|---------------|
| **Allowlists** | รายการขาวสำหรับคำยืม ชื่อเฉพาะ คำย่อที่ทราบ | สูงสำหรับช่องว่างที่ทราบ ศูนย์สำหรับช่องว่างที่ไม่ทราบ |
| **Fuzzy matching** | ยอมรับคำที่อยู่ภายในระยะแก้ไข 1 ของรูปแบบที่ทราบ | จับการพิมพ์ผิดและตัวแปรอักขรวิธีเล็กน้อย |
| **การให้คะแนนความเชื่อมั่น** | ถ่วงน้ำหนักผลลัพธ์ FST ตามความสมบูรณ์ของกระบวนทัศน์ | ต้องการข้อมูลเมตาการครอบคลุมกระบวนทัศน์ |
| **เกณฑ์เฉพาะหมวดหมู่** | เกณฑ์ที่แตกต่างกันสำหรับโดเมนที่แตกต่างกัน (การแพทย์อาจมีคำยืมมากกว่า) | ต้องการคลังข้อมูลที่แท็กตามโดเมน |
| **Allowlists ที่ชุมชนดูแล** | ผู้พูดส่งคำที่ FST ควรยอมรับ | ยั่งยืนที่สุดในระยะยาว; ต้องการโครงสร้างพื้นฐานการมีส่วนร่วมของชุมชน |

### 3.4 การวัดอัตรา

อัตราการปฏิเสธเท็จต้องวัดเชิงประจักษ์บนคลังข้อมูลตัวแทน:

1. นำคลังข้อมูลประโยค Cree ที่ทราบว่าถูกต้อง 500+ ประโยค (ตำราเรียน การแปลที่ผ่านการตรวจสอบ)
2. รันทุกคำผ่าน FST
3. สำหรับทุกคำที่ FST ปฏิเสธ ให้ผู้พูดสองภาษาจำแนก: คำที่ถูกต้องที่ FST พลาด หรือไม่ถูกต้องจริงๆ?
4. `false_rejection_rate = valid_but_rejected / total_valid_words`

การวัดนี้เป็นหนึ่งในการทดลองตรวจสอบที่จำเป็น (Scoring Spec §1.6)

## ส่วนที่ 4: ใครเป็นผู้ตัดสินว่าอะไรคือ "ถูกต้อง"?

### 4.1 มิติทางการเมืองของการประเมินผล

เมตริกการประเมินผลไม่ใช่เครื่องมือที่เป็นกลาง เมตริกทุกตัวฝังทฤษฎีว่า "การแปลที่ถูกต้อง" หมายความว่าอะไร และทฤษฎีนั้นมีผลกระทบ:

- FST ที่สร้างจากภาษา Cree วรรณกรรมจะลงโทษภาษา Cree ภาษาพูด นี่คือการเลือก *ทางการเมือง* เกี่ยวกับระดับภาษาใดที่มีคุณค่า
- คลาสตัวแปรที่ยอมรับรูปแบบสำเนียงหนึ่งแต่ไม่ยอมรับอีกรูปแบบหนึ่งทำให้ภาษาเป็นมาตรฐานโดยนัย การทำให้เป็นมาตรฐานเป็นการกระทำทางการเมืองที่มีประวัติศาสตร์อาณานิคมยาวนาน
- ตัวตรวจสอบความหมายที่ต้องการการทับซ้อน lemma แน่นอนลงโทษการถอดความเชิงสร้างสรรค์ — กลยุทธ์การแปลที่สำคัญที่นักแปลที่มีทักษะใช้โดยตั้งใจ

Microeval ทำให้การเลือกเหล่านี้ *ชัดเจน* คลาสตัวแปรทุกคลาส รายการ allowlist ทุกรายการ กฎความเท่าเทียมทางความหมายทุกกฎ เป็นการตัดสินใจที่แยกจากกัน มีเอกสาร และตรวจสอบได้ นี่คือคุณสมบัติ ไม่ใช่ข้อบกพร่อง: หมายความว่าชุมชนสามารถตรวจสอบ ท้าทาย และแก้ไขกฎที่ควบคุมวิธีการประเมินภาษาของตน

### 4.2 การกำกับดูแลชุมชนของกฎการประเมินผล

สำหรับภาษาพื้นเมืองโดยเฉพาะ การตัดสินใจการประเมินผลต้องอยู่ภายใต้การกำกับดูแลของชุมชนภาษา ไม่ใช่นักวิจัยหรือวิศวกรภายนอก นี่ไม่ใช่แค่หลักการทางจริยธรรม (แม้ว่าจะเป็นเช่นนั้น) — แต่เป็นข้อกำหนดความถูกต้อง มีเพียงผู้พูดที่คล่องแคล่วเท่านั้นที่สามารถกำหนดได้ว่าตัวแปรใดยอมรับได้

โมเดลการกำกับดูแล:

1. **นักวิจัยเสนอ** คลาสตัวแปร รายการ allowlist และกฎความหมายตามการวิเคราะห์ทางภาษาศาสตร์
2. **ผู้พูดตรวจสอบ** แต่ละข้อเสนอและอนุมัติ ปฏิเสธ หรือแก้ไข
3. **กฎที่ได้รับการอนุมัติ** ถูก commit ไปยัง codebase พร้อมการระบุแหล่งที่มาของผู้พูด
4. **กฎที่มีข้อพิพาท** ถูกตั้งค่าสถานะสำหรับการอภิปรายในชุมชน — ถูกยกเว้นจากการให้คะแนนจนกว่าจะได้รับการแก้ไข
5. **ชุมชนสามารถเพิกถอน** กฎใดก็ได้ตลอดเวลาโดยลบออกจากชุดที่ได้รับการอนุมัติ

โมเดลนี้ต้องการโครงสร้างพื้นฐาน (evaluation harness, การควบคุมเวอร์ชัน, โปรโตคอลการตรวจสอบของผู้พูด) และความสัมพันธ์ (ความไว้วางใจระหว่างนักวิจัยและสมาชิกชุมชน) การสร้างโครงสร้างพื้นฐานนี้เป็นส่วนหนึ่งของวิธีการ microeval

### 4.3 การแปรปรวนของสำเนียง

คำถามการกำกับดูแลที่ยากที่สุด: เมื่อสองสำเนียงของภาษาไม่เห็นด้วยเกี่ยวกับรูปแบบ อันไหน "ถูกต้อง"?

คำตอบของ Microeval: **ทั้งสองถูกต้อง** ตัวแปรสำเนียงถูกแทนเป็นคลาสตัวแปรเพิ่มเติมพร้อมแท็กสำเนียง คะแนนแบบผสมสามารถคำนวณต่อสำเนียงหรือข้ามสำเนียง ขึ้นอยู่กับสิ่งที่การประเมินผลพยายามวัด

สิ่งนี้ต้องการให้คลังข้อมูลถูกแท็กตามสำเนียงและคลาสตัวแปรต้องรับรู้สำเนียง นอกจากนี้ยังต้องการผู้พูดจากหลายสำเนียงเพื่อเข้าร่วมในการตรวจสอบ เอกสาร Corpus Partnership Strategy กล่าวถึงข้อกำหนดเหล่านี้

## ส่วนที่ 5: ความสัมพันธ์กับงานก่อนหน้า

### 5.1 สิ่งที่ Microeval ไม่ใช่

| การอ้างสิทธิ์ที่เราไม่ได้อ้าง | เหตุใด |
|------------------------|---------|
| "เมตริกสากลไม่มีประโยชน์" | เมตริกเหล่านั้นให้พื้นฐานที่จำเป็นและความสามารถในการเปรียบเทียบข้ามภาษา Microeval เสริม ไม่ใช่แทนที่ |
| "เมตริกประสาทไม่สามารถทำงานสำหรับ LRL" | ทำได้ — ด้วยการปรับแต่งบนข้อมูลเฉพาะภาษา แต่ข้อมูลนั้นแทบไม่มี Microeval ทำงานได้ *ตอนนี้* |
| "การประเมินผลด้วย FST เป็นสิ่งใหม่" | FST ถูกใช้ใน NLP มาหลายทศวรรษ ความใหม่คือการนำไปใช้อย่างเป็นระบบในฐานะเมตริกการประเมินผล MT |
| "LYSS ดีกว่า COMET" | เราไม่ทราบ — เรายังไม่ได้ทำการศึกษาความสัมพันธ์กับมนุษย์ เราเชื่อว่า LYSS มีข้อมูล *มากกว่า* สำหรับภาษาโพลีซินเทติก แต่เราไม่สามารถอ้างว่า *แม่นยำกว่า* จนกว่าเราจะมีหลักฐาน |

### 5.2 งานก่อนหน้าที่ใกล้เคียงที่สุด

| งาน | ความสัมพันธ์กับ Microeval |
|------|--------------------------|
| **MorphEval** (Sánchez-Cartagena & Toral, 2024) | การทดสอบเชิงเปรียบเทียบสำหรับปรากฏการณ์สัณฐานวิทยา — เสริมกัน MorphEval ทดสอบว่าระบบ *สามารถ* ผลิตสัณฐานวิทยาได้หรือไม่; LYSS ทดสอบว่าระบบ *ทำ* หรือไม่ |
| **CheckList** (Ribeiro et al., 2020) | วิธีการทดสอบพฤติกรรมสำหรับ NLP — กระบวนทัศน์ที่คล้ายคลึงกัน CheckList เป็นวิธีการ; microeval ก็เป็นวิธีการเช่นกัน ที่นำไปใช้กับการประเมินผลแทนการทดสอบ |
| **HyTER** (Dreyer & Marcu, 2012) | ตาข่ายความหมายเท่าเทียม — ขนานแนวคิดที่ใกล้เคียงที่สุดกับ LYSS-eq HyTER ระบุการถอดความ; LYSS-eq ระบุตัวแปรสัณฐานวิทยา HyTER ต้องการการสร้างตาข่ายด้วยตนเองต่อประโยค; กฎ LYSS-eq ใช้ทั่วทั้งคลัง |
| **การครอบคลุม Apertium** | ใช้การครอบคลุม FST เป็นตัวแทนสำหรับคุณภาพผลลัพธ์ MT — คาดการณ์ LYSS-fst โดยตรง ไม่ได้ทำให้เป็นทางการในฐานะเมตริกหรือรวมเข้ากับกรอบการให้คะแนน |
| **FUSE** (AmericasNLP 2025) | การประเมินผลตามคุณลักษณะสำหรับภาษาพื้นเมืองอเมริกา — ใกล้เคียงที่สุดในแง่จิตวิญญาณ FUSE เสนอคุณลักษณะทางภาษาศาสตร์เป็นมิติการประเมินผล; LYSS ใช้คุณลักษณะเฉพาะสำหรับภาษาเฉพาะ จำเป็นต้องมีการเปรียบเทียบแบบตัวต่อตัว |
| **AfriCOMET** (Wang & Adelani, 2024) | COMET ที่ปรับแต่งสำหรับภาษาแอฟริกา — แสดงให้เห็นว่าเมตริกประสาท *สามารถ* ปรับได้ Microeval เป็นส่วนเสริมแบบกฎสำหรับภาษาที่ไม่มีข้อมูลการปรับแต่ง |

### 5.3 ความแตกต่างหลัก

งานก่อนหน้าทั้งหมดในการประเมินผลที่รับรู้สัณฐานวิทยาไม่ว่าจะ:
1. **เสนอกรอบทั่วไป** โดยไม่ใช้งานสำหรับภาษาเฉพาะ (FUSE, CheckList)
2. **ใช้งานสำหรับภาษาที่มีทรัพยากรสูง** ที่มีข้อมูลฝึก (MorphEval มุ่งเน้นคู่ยุโรป)
3. **ปรับแต่งเมตริกประสาท** ซึ่งต้องการข้อมูลที่เราไม่มี (AfriCOMET)

Microeval ออกแบบมาเฉพาะสำหรับกรณีที่:
- เครื่องมือทางภาษาศาสตร์ (FSTs, พจนานุกรม) มีอยู่
- ข้อมูลสำหรับการปรับแต่งเมตริกประสาทไม่มี
- ความซับซ้อนทางสัณฐานวิทยาของภาษาทำให้เมตริกพื้นผิวล้มเหลว
- การประเมินผลต้องใช้งานได้ *ตอนนี้* ไม่ใช่หลังจากการรณรงค์รวบรวมข้อมูล

## ส่วนที่ 6: คำถามที่ยังเปิดอยู่และข้อจำกัดที่ซื่อสัตย์

### 6.1 คำถามที่ยังไม่ได้รับการแก้ไข

1. **เมตริก LYSS สัมพันธ์กับการตัดสินคุณภาพของมนุษย์หรือไม่?** เราไม่ทราบ การศึกษาความสัมพันธ์กับมนุษย์ที่จำเป็น (คู่ประโยค 200+ คู่, ผู้พูดสองภาษา 3+ คน) ยังไม่ได้ดำเนินการ จนกว่าจะดำเนินการ คะแนน LYSS เป็นการประมาณทางวิศวกรรม ไม่ใช่การวัดคุณภาพที่ตรวจสอบแล้ว

2. **เมตริก LYSS ทำงานอย่างไรเมื่อภาษาเปลี่ยนแปลง?** ภาษาที่มีชีวิตพัฒนา — คำยืมใหม่ สำเนียงที่เปลี่ยนแปลง คำใหม่ที่เกิดขึ้น FST และคลาสตัวแปรต้องได้รับการดูแล ภาระการดูแลคืออะไร? เราไม่ทราบ

3. **คุณภาพ FST ขั้นต่ำสำหรับ LYSS-fst ที่มีประโยชน์คืออะไร?** หาก FST ครอบคลุมเพียง 60% ของคำศัพท์ LYSS-fst ยังมีประโยชน์หรือไม่ หรือสัญญาณรบกวนท่วมสัญญาณ? เราต้องการหลักฐานเชิงประจักษ์

4. **Microeval สามารถทำงานสำหรับความท้าทายที่ไม่ใช่สัณฐานวิทยาได้หรือไม่?** ภาษาที่มีความแตกต่างของเสียง พยัญชนะคลิก หรืออักษรโลโกกราฟิกนำเสนอความท้าทายการประเมินผลที่ FST ไม่ได้กล่าวถึง Microeval อาจไม่ใช้ได้ — หรืออาจต้องการเครื่องมือที่แตกต่างกัน

5. **เราจะจัดการกับปัญหาการเริ่มต้นเย็นอย่างไร?** การสร้างชุด microeval ต้องการความเชี่ยวชาญทางภาษาศาสตร์ สำหรับภาษาที่ไม่มีชุมชนภาษาศาสตร์เชิงคำนวณที่ยังคงทำงานอยู่ ใครเป็นผู้ทำงาน?

### 6.2 ข้อจำกัดที่ซื่อสัตย์ของ LYSS

| ข้อจำกัด | ความรุนแรง | การบรรเทา |
|-----------|----------|-----------|
| ไม่มีข้อมูลความสัมพันธ์กับมนุษย์ | 🔴 วิกฤต | การทดลองตรวจสอบที่จำเป็น #1 |
| อัตราการปฏิเสธเท็จของ FST ยังไม่ได้วัด | 🔴 วิกฤต | การทดลองตรวจสอบที่จำเป็น #2 |
| ใช้งานสำหรับภาษาเดียวเท่านั้น (CRK) | 🟡 มีนัยสำคัญ | การพอร์ตภาษาที่สอง (North Sámi) วางแผนไว้ |
| คลาสตัวแปรอาจไม่สมบูรณ์ | 🟡 มีนัยสำคัญ | การตรวจสอบชุมชน + การเพิ่มอย่างต่อเนื่อง |
| ตัวตรวจสอบความหมายต้องการ spaCy | 🟡 มีนัยสำคัญ | การพึ่งพาที่เป็นทางเลือก; การลดระดับอย่างสง่างาม |
| การครอบคลุมพจนานุกรมส่งผลต่อคุณภาพ LYSS-sem | 🟡 มีนัยสำคัญ | ข้อกำหนดขนาดพจนานุกรมขั้นต่ำมีเอกสาร |
| ไม่สามารถตรวจจับความคล่องแคล่วหรือความเป็นธรรมชาติ | 🟡 มีนัยสำคัญ | ต้องการการประเมินผลของมนุษย์หรือเมตริกประสาท |
| ต้องการความเชี่ยวชาญทางภาษาศาสตร์เพื่อขยาย | 🟡 มีนัยสำคัญ | เอกสารวิธีการ (บทความนี้) ลดอุปสรรค |

### 6.3 เส้นทางข้างหน้า

> *"หากเรามุ่งเน้นเฉพาะสิ่งที่สรุปทั่วไปได้ เราจะลืมสิ่งที่ไม่ได้สรุปทั่วไปอย่างหลีกเลี่ยงไม่ได้ — และสูญเสียภาษาเหล่านี้พร้อมกับความรู้และภูมิปัญญาทั้งหมด"*

Microeval ไม่ใช่วิธีแก้ปัญหาการประเมินผล แต่เป็นแนวปฏิบัติ — วินัยในการใส่ใจสิ่งที่ทำให้แต่ละภาษาแตกต่าง และเข้ารหัสความใส่ใจนั้นในโค้ดที่ทำงานได้ แนวปฏิบัตินี้ต้องใช้แรงงาน เฉพาะเจาะจงกับภาษา และไม่มีวันเสร็จสิ้น แต่ผลิตสิ่งที่กระบวนทัศน์เมตริกสากลไม่สามารถทำได้: การประเมินผลที่พูดภาษาที่ตนประเมิน
