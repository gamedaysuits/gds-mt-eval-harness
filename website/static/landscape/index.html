The Gap Nobody Filled

There are benchmarks, models, and eval tools — but nothing that ties them together for the languages that need MT most.

📊

Benchmarks exist. Harnesses don't.

FLORES-200 and Tatoeba provide test data. But there's no open framework to run arbitrary translation methods — LLM, NMT, pipeline, hybrid — against the same data and compare them fairly. You train your model, score it yourself, and submit a number to a markdown file.

🔤

Metrics are blind to morphology.

chrF and BLEU count character/word overlaps. COMET uses XLM-R embeddings trained mostly on European languages. None of them can tell you whether nikî-nipân is a structurally valid Plains Cree word. For polysynthetic languages, structural validity is the first question — and no metric asks it.

🔐

Open-source ≠ sovereignty.

Tatoeba is CC-BY. FLORES is Meta-controlled. Neither framework has a concept of community-owned data, encrypted test sets, or governance-org key custody. OCAP principles and open benchmarking are structurally incompatible — unless you design for sovereignty from the ground up.

⚡

Champollion connects all of this.

Not a competitor to WMT or Tatoeba — a catalyst. One framework that plugs into existing benchmarks, eval tools, and methods. Test any translation approach against community-governed data, verify with morphological validation and human judgment, deploy to production. For any language pair, any method type.
